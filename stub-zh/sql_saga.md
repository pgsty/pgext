`sql_saga` 增加有效时间时代、时间唯一性与外键、可更新的当前/局部视图、系统版本历史，以及集合式时间合并操作。版本 `1.0` 面向 PostgreSQL 18，并使用原生 `WITHOUT OVERLAPS` 约束。实体需要保留互不重叠的历史版本而不仅是最新一行时，可使用该扩展。

### 核心流程

安装 `btree_gist` 依赖，定义基于范围的时间主键，再把表登记为 era：

```sql
CREATE EXTENSION btree_gist;
CREATE EXTENSION sql_saga;

CREATE TABLE legal_unit (
  id integer NOT NULL,
  legal_ident text NOT NULL,
  name text NOT NULL,
  valid_range daterange NOT NULL,
  PRIMARY KEY (id, valid_range WITHOUT OVERLAPS)
);

SELECT sql_saga.add_era(
  'legal_unit'::regclass,
  'valid_range',
  'valid'
);

SELECT sql_saga.add_unique_key(
  table_oid => 'legal_unit'::regclass,
  column_names => ARRAY['legal_ident'],
  key_type => 'natural',
  unique_key_name => 'legal_unit_legal_ident_valid'
);

SELECT sql_saga.add_current_view('legal_unit'::regclass);
```

有效期使用左闭右开 `[)` 区间。当前行是有效期包含当前日期/时间的行，因此也包括结束时间已知但仍在未来的行。

### 主要 API

- `add_era(...)` / `drop_era(...)`：登记或删除时间元数据、检查与同步触发器。
- `add_unique_key(...)`：增加自然键、主键或带谓词的时间唯一性。
- `add_foreign_key(...)`：检查参与表并创建普通表到时间表或时间表之间的外键，默认同时创建辅助索引。
- `add_current_view(...)`：通过可更新视图暴露当前行；默认删除行为是截断有效期，而不是物理擦除历史。
- `add_for_portion_of_view(...)`：提供 SQL:2011 风格的指定有效期片段更新。
- `temporal_merge(target_table, source_table, ...)`：从暂存表规划并执行集合式插入、更新、拆分、合并和可选删除。
- `add_system_versioning(...)`：除有效时间外再创建事务时间历史。

`temporal_merge` 默认采用非破坏性的 `MERGE_ENTITY_PATCH`。破坏性行为由独立的 `delete_mode` 控制；默认要求源表有唯一且可排序的 `row_id` 字段。过程会创建会话本地的 `pg_temp.temporal_merge_plan` 与 `pg_temp.temporal_merge_feedback` 表，并在事务结束时删除。

### 注意事项

时间历史需要稳定实体标识符与时间范围共同组成 `WITHOUT OVERLAPS` 复合键。简单主键或 `GENERATED ALWAYS AS IDENTITY` 会阻止保存同一实体的多个历史版本；应使用兼容复合键，必要时采用 `GENERATED BY DEFAULT`。扩展会创建约束、索引、触发器、视图、辅助函数与元数据，因此应以表所有者执行登记并审查所有生成对象。

`temporal_merge` 是 `SECURITY INVOKER`，表权限和 RLS 仍然生效。避免在同一会话的多次调用之间切换角色，因为临时表保留创建者所有权。调试 GUC `sql_saga.temporal_merge.log_plan`、`.log_feedback`、`.log_sql` 会把计划、逐行反馈或生成 SQL 写入服务器日志。破坏性合并/删除模式只能先在事务隔离的副本上演练后再使用。
