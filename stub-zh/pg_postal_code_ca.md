## 用法

来源：

- [官方 README](https://github.com/rlichtenwalter/pg_postal_code_ca/blob/master/README.md)
- [官方 1.0 版 SQL](https://github.com/rlichtenwalter/pg_postal_code_ca/blob/master/pg_postal_code_ca--1.0.sql)
- [官方 control 文件](https://github.com/rlichtenwalter/pg_postal_code_ca/blob/master/pg_postal_code_ca.control)

`pg_postal_code_ca` 提供紧凑、可验证、规范化输出并支持 B-tree 排序的加拿大邮政编码类型。权威的版本 `1.0` SQL 把类型命名为 `postal_code_ca`；上游 README 仍使用旧名 `canadian_postal_code`，不能原样照抄。

### 核心流程

对于必须拒绝畸形邮编并统一显示格式的字段，可使用该类型：

```sql
CREATE EXTENSION pg_postal_code_ca;

CREATE TABLE locations (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  postal_code postal_code_ca NOT NULL
);

INSERT INTO locations (postal_code) VALUES ('h2e3f1');

SELECT postal_code
FROM locations
WHERE postal_code >= 'H2E 3F0'::postal_code_ca
ORDER BY postal_code;
```

不带空格的输入会被规范化为大写标准形式，例如 `H2E 3F1`。字母/数字位置无效时，类型转换会报错。

### 对象

- `postal_code_ca`：带输入/输出函数的定长 C 类型。
- `<`、`<=`、`=`、`<>`、`>=`、`>`：比较运算符。
- `postal_code_ca_cmp(postal_code_ca, postal_code_ca)`：三路比较辅助函数。
- `postal_code_ca_ops`：默认 B-tree 运算符类，支持有序索引、等值查询、范围谓词和排序。

### 注意事项

该类型验证扩展实现的加拿大邮编语法，但不能证明邮编已分配、可投递或仍然有效。如果需要验证存在性，应再查询权威地址数据源。

README 与类型名不一致源于上游重命名：应使用安装 SQL 中的 `postal_code_ca`，迁移前检查现有模式。比较顺序是扩展定义的邮编字节顺序，不代表地理邻近度或投递距离。
