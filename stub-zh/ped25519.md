ped25519 0.1.0 版以内嵌 Rust 函数提供 Ed25519 签名和验证。签名函数接收消息字节、32 字节公钥和 32 字节私钥；验证函数接收消息字节、64 字节签名和 32 字节公钥。

### 签名与验证

下面使用上游公开的固定示例密钥，而不是生产密钥：

```sql
CREATE EXTENSION ped25519;

WITH keys AS (
    SELECT
        decode('NZV4l8hck3iUqInENyI+nn5vkW7rqzQg0uiuuZkPnHE=', 'base64') AS public_key,
        decode('eT7qaT8vkIgCl6/9EmEDYYEgxA0oOgHc0P6UYzcQN28=', 'base64') AS private_key
), signed AS (
    SELECT
        convert_to('some data for signing', 'UTF8') AS message,
        public_key,
        ed25519_sign(
            convert_to('some data for signing', 'UTF8'),
            public_key,
            private_key
        ) AS signature
    FROM keys
)
SELECT ed25519_verify(message, signature, public_key)
FROM signed;
```

结果应为 true。ped25519 不生成密钥，也不提供 key store；密钥生成、轮换、备份和访问控制仍由应用负责。

### 安全与兼容性

不要把真实私钥放进 migration 文件、可被监控看到的查询文本或参数、以及应用日志中。应撤销 public execute，只把签名权限授予范围很窄的角色；验证权限通常可以单独授予。若能使用专用 key manager 或硬件保护密钥，应优先在数据库外完成签名。

Rust 实现在完整返回所有长度错误前，就会对传入的公钥和组合密钥 buffer 进行切片。应强制检查精确输入长度，避免进入 panic/error 路径。审阅的 pgrx 构建声明支持 PostgreSQL 14 至 16 feature，并要求超级用户安装；需要在确切目标主版本上验证软件包。
