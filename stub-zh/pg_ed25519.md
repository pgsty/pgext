`pg_ed25519` 为 `bytea` 和 `text` 消息提供 `ed25519.sign` 与 `ed25519.verify` 重载。签名接受 32-byte 公钥和 32-byte 私钥种子，返回 64-byte 签名；验证需要消息、签名和公钥。

### 签名与验证

```sql
CREATE EXTENSION pg_ed25519;

WITH keys AS (
  SELECT
    decode('NZV4l8hck3iUqInENyI+nn5vkW7rqzQg0uiuuZkPnHE=', 'base64') AS public_key,
    decode('eT7qaT8vkIgCl6/9EmEDYYEgxA0oOgHc0P6UYzcQN28=', 'base64') AS private_key
), signed AS (
  SELECT public_key,
         ed25519.sign(text 'some data for signing', public_key, private_key) AS signature
  FROM keys
)
SELECT ed25519.verify(text 'some data for signing', signature, public_key)
FROM signed;
```

示例密钥对是上游公开测试材料，绝不能重复使用。

### 注意事项

- 该扩展不是密钥生成器或密钥库。通过 SQL 传递私密材料可能使其暴露在应用日志、语句日志、监控或 `pg_stat_activity` 中；真实秘密应使用受控签名基础设施。
- 版本 `0.2` 内置自己的 Ed25519 和 SHA-512 C 实现。在互操作或安全用途前，应使用维护中的密码库独立验证签名。
- 上游声称支持 PostgreSQL 9.4 及以上版本，但没有当前兼容性或安全维护声明。不可变函数声明只是规划器元数据，不是秘密处理保证。
