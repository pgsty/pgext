`pg_ed25519` 0.2 在固定的 ed25519 schema 中提供 Ed25519 签名和验证函数。它接受原始的 32 字节公钥与私钥，并产生 64 字节签名。它只提供密码学操作，不负责密钥生成、保存、轮换或访问控制。

### 核心流程

在 PostgreSQL 外部生成密钥，按上游说明保留原始 32 字节密钥材料，并以绑定的 bytea 参数传入，不要把它写成 SQL 字面量。

```sql
CREATE EXTENSION pg_ed25519;

PREPARE sign_message(bytea, bytea, bytea) AS
SELECT ed25519.sign($1, $2, $3);

PREPARE verify_message(bytea, bytea, bytea) AS
SELECT ed25519.verify($1, $2, $3);
```

签名参数顺序为消息、公钥、私钥；验证参数顺序为消息、签名、公钥。

### 函数索引

- `ed25519.sign(bytea, bytea, bytea)` 及其 text 消息重载返回 64 字节签名。
- `ed25519.verify(bytea, bytea, bytea)` 及其 text 消息重载返回 boolean。
- 这些函数为 immutable、strict 和 parallel-safe；密钥或签名长度错误时会报错。

### 安全与互操作性

- 私钥字节会跨过 SQL 边界。应避免其进入查询文本、日志、统计、追踪、结果采集，或被可查看会话的角色读取；无法接受这种暴露面时应在应用侧签名。
- text 重载签署服务器编码后的精确文本字节。需要让其他系统复现签名时，应使用经过明确规范化编码的 bytea。
- 扩展安装到固定的 ed25519 schema 且不可迁移。调用时应限定 schema，并严格控制 EXECUTE 权限。
